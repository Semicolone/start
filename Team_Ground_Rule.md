# 팀 그라운드 룰

# 1. 팀 목표

- 졸업 프로젝트를 **성실하게 수행하여 완성도 높은 결과물을 제출**하는 것을 목표로 한다.
- 팀원 모두가 각자의 역할을 책임지고 수행하며 **협업 경험을 쌓는 것**을 중요한 가치로 둔다.
- 일정 내 **MVP 개발 및 시연 가능한 수준의 결과물 완성**을 목표로 한다.

---

# 2. 커뮤니케이션 규칙

### 1) 연락 및 응답

- 팀 채널: **카카오톡**
- 메시지 확인 후 **24시간 이내 응답**
- 급한 사안은 **멘션 또는 전화**

### 2) 회의

- 정기 회의: **주 1~2회**
- 회의 전 **아젠다 공유**
- 회의 후 **회의록 작성 및 공유**

### 3) 정보 공유

- 모든 자료는 노션에 정리
- 개인적으로만 알고 있는 정보가 없도록 한다.

---

# 3. 역할과 책임

### 1) 역할 분담

각 팀원은 담당 역할을 명확히 가진다.

- **PM / 기획 → 업무 분담**
- **Frontend**
- **Backend**
- **Design → 지원**

※ 필요 시 서로 **서포트 가능**

### 2) 책임 원칙

- 맡은 업무는 **마감 전에 완료**
- 일정 변경 시 **미리 팀원에게 공유**
- 개인 사정 발생 시 **최대한 빨리 공유**

---

# 4. 일정 관리

### 1) 작업 관리

- 작업 관리는 **Notion / Jira / Trello / Github Issue** 사용
- 모든 작업은 **Task 단위로 관리**

### 2) 마감

- 개인 작업은 **팀 마감보다 최소 하루 전 완료**
- 지연될 경우 **사전에 공유**

---

# 5. 개발 협업 규칙

### 1) 코드 관리

- GitHub 사용
- **main 브랜치 직접 커밋 금지**
- **PR (Pull Request) 후 merge**

### 2) 코드 리뷰

- 최소 **1명 이상 리뷰 후 merge**

### 3) 문서화

- 주요 기능은 **README 또는 Notion에 문서화**

---

# 6. 참여 태도

- 회의 **무단 불참 금지**
- 팀 프로젝트 기간 동안 **최소한의 참여 기준 유지**
- 서로에 대한 **존중과 배려 유지**

---

# 7. 문제 발생 시 대응

문제가 발생할 경우 아래 순서로 해결한다.

1. 팀 내부 논의
2. 해결이 어려울 경우 **지도교수 또는 조교 상담**
3. 지속적인 미참여 시 **팀 차원의 공식 경고**
